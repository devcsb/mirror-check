🪞 Mirror Check

"도움됐어요" 같은 버튼이 실제 서버 집계인지, 사용자가 자기 거울을 보고 있는 것(로컬 저장)에 불과한지 한 번에 판정합니다.

🪞 Mirror Check (드래그해서 북마크바에 놓기)
이 링크를 북마크바로 드래그하세요. 이미 북마크가 있고 소스를 수정했다면 다시 드래그해 갱신하세요.

사용법

  1. 검사할 페이지로 이동합니다. 예: tael.github.io/interior-guide
  2. 북마크바의 🪞 Mirror Check를 클릭합니다. 우하단에 패널이 뜹니다.
  3. 의심스러운 버튼(예: "도움됐어요")을 클릭합니다.
  4. 판정을 확인합니다:
    • FAKE localStorage/cookie만 변하고 네트워크 요청이 없음 → 가짜
    • REAL 서버 요청 발생, 로컬 저장 없음 → 실제 서버 집계
    • MIXED 둘 다 발생 → 서버 집계 + 로컬 캐시(중복 방지용)
  5. 다시 클릭하면 패널이 닫힙니다.

작동 원리 (3줄 요약)

  1. Storage.prototype.setItem, document.cookie setter를 monkey-patch해 로컬 쓰기를 기록합니다.
  2. fetch, XMLHttpRequest.open, navigator.sendBeacon을 래핑해 네트워크 호출을 기록합니다.
  3. 클릭 직후 2.5초 창(window) 내 로그를 비교해 verdict를 결정합니다.

한계

  • 서버 요청이 Service Worker를 거치거나 WebSocket으로 전송되면 탐지되지 않을 수 있습니다.
  • 버튼 클릭 후 디바운스(수 초 지연) 후 배치 전송하는 경우 감지창을 놓칠 수 있습니다 — 필요 시 코드의 windowMs를 늘리세요.
  • 엄격한 CSP(Content-Security-Policy)가 걸린 페이지에서는 북마클릿 실행이 차단될 수 있습니다.
